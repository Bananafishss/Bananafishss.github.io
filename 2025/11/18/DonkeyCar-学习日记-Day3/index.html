DonkeyCar 经典视觉检测总结

Day 3 - 霍夫变换车道线检测方法若干问题的改进

1. 实验流程

本章介绍整个感知算法流水线（pipeline）的技术细节，包括透视变换、颜色过滤、ROI、聚类、拟合和几何计算。

1.1 BEV 透视变换（Bird’s Eye View）

摄像头存在透视畸变：近处宽、远处窄

使用 单应性矩阵 Homography Matrix 将图像从原始视角映射到 鸟瞰视角 BEV

需要四个 Source Points（梯形）与四个 Destination Points（矩形）

得到转换矩阵 M = cv2.getPerspectiveTransform(src, dst)

cv2.warpPerspective(img, M, size) 得到 BEV 图像

效果： BEV 图像中的左右车道线趋于平行，更适合后续几何分析。

1.2 颜色过滤（HSV Filtering）

将 BGR → HSV 空间（Hue、Saturation、Value）

HSV 过滤比灰度阈值法更不受光照影响

本实验主要保留白色线条，同时剔除黄色线条与反光区域

过滤逻辑：

白色：低饱和度 + 高亮度

黄色：中等饱和度 + 特定 Hue 范围，直接剔除

最终得到仅包含白色车道线的二值图像。

1.3 ROI 掩膜（Region of Interest）

去除天空、桌椅、道路外区域

使用五边形 ROI，上窄下宽，符合车辆视野形状

ROI 顶点根据图像比例动态计算

与二值图像做按位与 → 仅保留小车前方区域

好处：

大幅减少误检

降低计算量

提高左右车道的提取精度

1.4 车道区域筛选（Lane Region Filtering）

即便经过 ROI，仍可能有噪声或非车道区域

对每一行（y）统计横向亮度分布

自动估计车道的大致横向范围

动态带宽掩膜：

底部窄，上部宽

模拟透视变化，让算法能捕捉远处细小车道线

最终只保留主要车道区域，提高后续聚类成功率。

1.5 温和清理与线段拼接（Soft Cleaning & Merging）

(1) 温和清理

使用连通域分析

删除：

面积太小的噪点

横向过宽的块

保留：

合理纵向线条

(2) 拼接断裂线段

对左右两半图像进行形态学闭操作（closing）

补全车道线间的小断裂

结果为：干净、连续、可检测的左右两条车道线。

1.6 骨架化 + DBSCAN 聚类

处理流程：

Canny 边缘检测

闭操作

骨架化（Thinning） → 得到单像素宽度的主干线

DBSCAN 聚类 → 自动区分左右车道线

不需要指定簇数

噪声（label = -1）自动剔除

小簇（<50点）过滤

Silhouette Score 评估聚类质量

优点：

抗噪能力强

能适应车道线断裂、光照变化

自动找到左右两条主干线

1.7 曲线拟合（Curve Fitting）

对每个簇的点集进行二次多项式拟合：

x = a·y² + b·y + c

为什么不用 y = f(x)？

在弯道可能出现“一条 y 对应多个 x”的多值问题

而 y 始终单调递增，更适合建模 BEV 中的纵向车道结构

输出：

左车道拟合曲线

右车道拟合曲线

在 BEV 图像上叠加可视化

1.8 坐标生成、偏向角、中心偏移量（Lane Point Generation）

完成拟合后，需要将连续曲线离散化成像素点：

从 y = ymin → ymax 均匀采样

根据函数计算 x

得到结构化点集（按 y 递增排序）

额外计算：

两条车道线的方向角（朝向角）

左右车道线与中线之间的距离

小车相对中心线的偏移量

所有结果存储到 JSON，用于控制模块。

未来可通过真实赛道标定：

将 pixel → meter 转换为真实距离

2. 结果分析（Results）

基于 111 张模拟器图像的测试结果：

a) 时间性能

平均耗时：0.015 s/帧

标准差：0.00725 s

最快：0.009 s

最慢：0.0397 s

➡ 支持高刷新率实时控制

b) 鲁棒性

识别准确率：93.69%

c) 混淆情况

在仅看到一条车道线时仍可保持区域识别

左右簇未混淆比例：98.19%

聚类数量：

平均：2.1

标准差：0.3

最优：2

最差：3

3. 结论（Conclusion）

整体算法在模拟器测试中表现稳定：

实时性强：支持高频控制循环

准确率高：93.69%

左右车道分离效果良好：98.19% 未混淆

能够处理车道丢失、部分遮挡等情况

适用于真实环境和模拟环境的车道检测任务。

4. 展望（Perspectives）

当前方案的局限：

对光照（阴影、反光）较敏感

参数（阈值、ROI等）依赖人工调整

在复杂环境的泛化能力有限

未来改进方向：

引入深度学习车道检测模型

社区成熟模型丰富

更强的抗光照、抗噪声鲁棒性

泛化能力明显优于传统方法

自适应阈值 / 滤波方法（但计算量较大）

深度模型 + 当前几何管线结合

进一步提高稳定性与实时性能
