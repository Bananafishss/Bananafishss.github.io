C/C++ 学习日记 类，实例，属性和方法

2.C++ 类，实例，属性和方法

类的定义

和结构体类似。声明如下：
‘’’cpp
class NomClasse {…};
‘’’

类的属性和方法

简而言之，属性（Attributes）就是类里的变量，方法（Méthodes）就是类里的成员函数。
声明一个类的方式如下：
‘’’cpp
class Student {
public:
string name;//属性
int age;
int grade;

void introduce() {       // 方法
    cout << "I am " << name << ", age " << age << endl;
}

void study() {           // 方法
    cout << name << " is studying..." << endl;
}

};
‘’’
调用：
‘’’cpp
s.introduce();
s.study();
‘’’
注意事项：

  1. 属性不需要像参数一样写在方法括号里，因为方法已经自动能访问它们。
    ‘’’cpp
    class A {
    public:
    int x;
    void print() const { // 直接访问属性,不需要另外在（）里写x
    cout << x;
    }
    };’’’

  2. 为了结构清晰，最好把类的实现和类的声明分开，声明放在h文件，实现放在cpp文件里。

  3. 在类外写方法时，编译器不知道这个函数属于哪个类，要用 :: 来告诉它。比如：
    ‘’’cpp
    class Car {
    public:
    int surface(int w, int h);
    };
    ///然后在类外实现时加上”::”：
    int Car::surface(int w, int h) {
    return w * h;
    }
    ‘’’

  4. 有些方法会修改对象的属性，有些不会。不会修改属性的方法叫 const 方法。要写成： retour nomMethode(…) const. const 放在括号后面表示“我保证不改对象”。
    ‘’’cpp
    int getX() const; // const 方法（不改属性）
    void setX(int x); // 普通方法（能改属性）
    ‘’’

const的四种用法

  1. ‘’’cpp
    const int i
    ‘’’
    对于变对象或属性来讲，这个形式表示该变量不可被修改。
  2. 对于方法来说，有三种情况：
    ‘’’cpp
    CONST1 type_de_retour methode(CONST2 parametre) CONST3;
    ‘’’
    CONST1表示锁返回方法，CONST2表示函数不能改变输入的参数值，CONST3表示该方法不能修改当前实例的属性。

类的成员的可见性

  1. public = 大家都能用（公开的）

  2. private 成员只能在类内部被访问（即：类自己的方法能用），类外面（包括实例化后的对象）不能直接访问。
    同一类的不同对象之间可以直接相互赋值，不需要借助getter，setter!
    举个例子：
    ‘’’cpp
    class A {
    private:
    int secret; // 私有属性
    public:
    void setSecret(int x) {
    secret = x; // ✔ 可以，类内部访问 secret
    }

    int getSecret() const {
    return secret; // ✔ 类内部访问 OK
    }
    };
    //Getter和Setter方法：从外部获取private成员

    int main() {
    A obj;

    obj.secret = 10; // X 错误！外部不能访问 private
    obj.setSecret(10); // ✔ 正确，通过 public 方法间接操作 private
    }
    ‘’’

构造函数和析构函数

  1. Constructeur

    构造函数的特点

    1. 名字和类名完全一样：Rectangle::Rectangle(…)
    2. 没有返回值（甚至不能写 void）。
    3. 需要创建对象时调用：Rectangle rect1(3.0, 4.0);
      举例：
      ‘’’cpp
      class Rectangle {
      public:
      double largeur;
      double hauteur;

      Rectangle(double l, double h) {
      largeur = l;
      hauteur = h;
      cout << “Constructeur appelé!” << endl;
      }
      };
      //在类声明外写构造函数（大部分情况写在类里面的public部分 写在外面需要加Rectangle::）：
      Rectangle::Rectangle(double hauteur, double largeur) {
      this->largeur = largeur;
      this->hauteur = hauteur ;
      }
      //当我创建对象时：
      Rectangle r(3.0, 4.0);
      //对象通过构造函数被自动创建。编译器内部会做：
      //1.创建一个 Rectangle 类型的对象 r.
      //2.把 r 的地址传给构造函数的 this 指针
      ‘’’
  2. Destructeur

    析构函数的特点

    1. 名字是 ~类名
    2. 没有返回值
    3. 对象消失时自动调用
      举例：
      ‘’’cpp
      class Rectangle {
      public:
      ~Rectangle() {
      cout << “Destructeur appelé!” << endl;
      //析构函数没有复杂功能时也可以不写 → 系统自动生成。
      }
      };
    4. 注意：写析构函数时只有那些用new创建的指针(data=nullptr;),数组(delete[] data;)需要初始化,那些int之类的变量存在栈里 自动被删除。

构造函数里的初始化列表

是函数里赋值的上位替代，速度更快，效率更高！
调用初始化列表的构造函数的方式和函数里赋值的方式完全一致如’Point(1, 0);’
初始化列表 = 在对象创建那一刻直接把属性初始化，而构造函数里赋值是将已经被初始化为随机值的值进行赋值，是第二次赋值。
必须用初始化列表的情况：遇到 const、引用、对象成员时。
模板：
‘’’cpp
ClassName::ClassName(type a, type b)//头文件里带参数的函数内初始化 和 初始列表初始化的声明都是这么写
: attribute1(a), attribute2(b), attribute3(5)//
{
// 构造函数体
}
‘’’
举例：
‘’’cpp
//类里（只声明，不写初始化列表）：
class Rectangle {
public:
Rectangle(double H, double L); // 这里只是声明
};
//类外（真正定义 + 初始化列表）：
Rectangle::Rectangle(double H, double L)
hauteur(H), largeur(L)
{}
‘’’

默认生成的构造函数

默认构造函数 = 一个“没有参数”的构造函数或者所有参数都有默认值的构造函数。
当你声明对象但不给它初始值时，就会调用它。

  1. 没有参数的构造函数：
    ‘’’cpp
    class Rectangle {
    public:
    Rectangle() { // ← 默认构造函数
    largeur = 1;
    hauteur = 1;
    }
    };
    //调用
    Rectangle r; // ← 这里会自动调用 Rectangle()
    ‘’’

  2. 所有参数都有默认值的构造函数、
    ‘’’cpp
    class Rectangle {
    public:
    Rectangle(double h = 1, double l = 1) { // 每个参数都有默认值
    hauteur = h;
    largeur = l;
    }
    };
    //调用
    Rectangle r; // 调用 Rectangle(1,1)
    Rectangle r2(5); // 调用 Rectangle(5,1)
    Rectangle r3(3,4); // 调用 Rectangle(3,4)

    ‘’’

  3. 默认构造函数的作用：

    1. 有时你想先创建对象，之后再设置属性

    2. 有时对象是另一个对象的成员，需要自动创建

    3. 有时你需要创建一个对象数组

复制构造函数

复制构造函数 = 用一个已经存在的对象，复制出一个新对象的“特殊构造函数”。
模板：
‘’’cpp
ClassName(const ClassName& other){
// 用 other 的内容初始化当前对象
}
‘’’
举例:
‘’’cpp
Rectangle::Rectangle(const Rectangle& r) {
this->largeur = r.largeur;
this->hauteur = r.hauteur;
}
‘’’

使用复制构造函数的场景：

1. 用一个对象初始化另一个对象
2. 把对象作为参数传给函数（按值传递）
3. 函数按值返回对象
4. 容器操作（如 vector）

必须自己写复制构造函数的场景：

当类里面有“指针”或“动态内存”时！
'''cpp
    class A {
    int* data;
};
//不能直接A a1;A a2 = a1;，会报错
//必须：
    A::A(const A& other) {
    data = new int(*other.data); // 深拷贝
}
'''

static成员

static 成员对每个该类的对象共享同一份 static 数据或 static 方法。
换句话说，它是同类下所有对象的全局变量或函数。

静态成员变量/属性

static 属性是全类共享的，不属于某一个对象，所以不应该放在“对象出生时执行的构造函数”里初始化。
‘’’cpp
class A {
public:
static int count;
};
//本例子中所有 A 的对象共享 同一个 count，不管创建多少个对象，count 就那一份
‘’’

C++中的异常处理

所谓的异常就是当发生特定事件时，程序执行被中断，并将控制权转移给名为管理器的特殊函数的过程。
异常处理分成两步：

  1. throw = 抛出异常（告诉coder出错了）
    ‘’’cpp
    throw “Erreur !”;
    throw 404;
    throw std::runtime_error(“Bad value”);
    ‘’’
    当程序执行到 throw，它会:
    停止当前代码
    跳到最近的 try/catch
    把错误信息送给 catch
  2. try+catch = 接住异常（怎么补救？）
    try 表示“这里可能会出错，我要小心执行”。
    catch 接住 throw 抛出的异常，并进行处理。
    可以捕获各种类型：
    字符串，int，自定义类对象，std::exception
    ‘’’cpp
    try {
    cout << “Avant erreur” << endl;
    throw “Problème !”; // ← 抛出异常
    cout << “Après erreur” << endl; // 永远不会执行
    }
    catch(const char* e) { // ← 捕获异常
    cout << “Exception capturée : “ << e << endl;
    }
    ‘’’
    流程：
  3. try 内代码执行.
  4. 遇到 throw → 直接跳到 catch.
  5. catch 接住错误并处理.
