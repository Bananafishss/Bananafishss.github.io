霍夫变换改进

  1. 改变ROI从梯形到五边形
  2. 预处理图像从HLS二值转换变成灰度值转换
  3. 用K-means和DBSCAN算法处理霍夫变换得到的线段集
  4. 增加BEV鸟瞰图变换
    def bev_perspective(img): h, w = img.shape[:2] src_points = np.float32([ [w * 0.05, h * 0.7], [w * 0.95, h * 0.7], [w * 0.70, h * 0.45], [w * 0.30, h * 0.45] ]) dst_points = np.float32([ [w * 0.25, h], [w * 0.75, h], [w * 0.75, 0], [w * 0.25, 0] ]) M = cv2.getPerspectiveTransform(src_points, dst_points) return cv2.warpPerspective(img, M, (w, h))
    本函数把原图中的梯形映射成矩形。

车道线相关Python内容

  2. Path对象用来创建和保存文件非常方便 在我的程序中使用Path对象存放输入输出图片。
  3. gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)把图片从RGB换成灰度空间的函数。
  4. blur = cv2.GaussianBlur(gray, (GAUSS_KSIZE, GAUSS_KSIZE), 0)对图片进行高斯滤波。
  5. v = np.median(blur)median函数是对矩阵求中位数。
  6. 将形如(x,y)的点集的x 和y提取出来的方法：首先有点集pointspoints = [(100, 720), (110, 680), (120, 640), (130, 600)]

DonkeyCar 学习日记

Day 1 - 霍夫变换车道线检测方法

🚗 思路流程整理

  1. 将 RGB 图像转换为 HLS 颜色空间，提取白色可能区域
  2. 阈值过滤后得到黑白二值图
  3. 设计 ROI 区域避免无关干扰
  4. 使用 Canny 边缘检测提取道路边缘
  5. 使用 HoughLinesP 检测直线
  6. 根据斜率判断左右车道线
  7. 根据路况选择拟合方式（直线/曲线）

🔍 Canny 边缘检测

参数说明：

参数 含义
image 输入图像（单通道灰度图）
threshold1 Canny低阈值
threshold2 Canny高阈值
apertureSize Sobel算子大小（3/5/7）
L2gradient True 更精确但更慢

📐 Hough 直线检测

作用：霍夫变换将边缘点转为参数空间，用投票机制拟合出直线。

➗ 区分左右车道线逻辑

通过霍夫拟合出的直线斜率：

  • 斜率 < 0 → 左车道线
  • 斜率 > 0 → 右车道线

🛣️ 判断直道还是弯道

类型 特征
直道 霍夫线段斜率接近
弯道 拟合出的线段呈弧线，斜率变化大

若是弯道，需要使用二次曲线拟合。

❗ 当前遇到的问题

  1. 很难准确区分真实白线与光照高亮区域
  2. 强反光区域容易被当成车道线
  3. 二值图噪声较多，导致霍夫检测不稳定
  4. 由于前一步二值化不够稳定，Canny 检测受噪声干扰较大

✅ 下一步我计划尝试：

  • 改进二值化方法，尝试转换到灰度空间
  • 修改ROI区域的形状使其更贴合实际形状
  • 尝试使用clustering，K聚簇，DBSCAN算法来进行曲线拟合
