DonkeyCar 经典视觉检测总结

Day 3 - 霍夫变换车道线检测方法若干问题的改进

1. 实验流程

本章介绍整个感知算法流水线（pipeline）的技术细节，包括透视变换、颜色过滤、ROI、聚类、拟合和几何计算。

1.1 BEV 透视变换（Bird’s Eye View）

摄像头存在透视畸变：近处宽、远处窄

使用 单应性矩阵 Homography Matrix 将图像从原始视角映射到 鸟瞰视角 BEV

需要四个 Source Points（梯形）与四个 Destination Points（矩形）

得到转换矩阵 M = cv2.getPerspectiveTransform(src, dst)

cv2.warpPerspective(img, M, size) 得到 BEV 图像

效果： BEV 图像中的左右车道线趋于平行，更适合后续几何分析。

1.2 颜色过滤（HSV Filtering）

将 BGR → HSV 空间（Hue、Saturation、Value）

HSV 过滤比灰度阈值法更不受光照影响

本实验主要保留白色线条，同时剔除黄色线条与反光区域

过滤逻辑：

白色：低饱和度 + 高亮度

黄色：中等饱和度 + 特定 Hue 范围，直接剔除

最终得到仅包含白色车道线的二值图像。

1.3 ROI 掩膜（Region of Interest）

去除天空、桌椅、道路外区域

使用五边形 ROI，上窄下宽，符合车辆视野形状

ROI 顶点根据图像比例动态计算

与二值图像做按位与 → 仅保留小车前方区域

好处：

大幅减少误检

降低计算量

提高左右车道的提取精度

1.4 车道区域筛选（Lane Region Filtering）

即便经过 ROI，仍可能有噪声或非车道区域

对每一行（y）统计横向亮度分布

自动估计车道的大致横向范围

动态带宽掩膜：

底部窄，上部宽

模拟透视变化，让算法能捕捉远处细小车道线

最终只保留主要车道区域，提高后续聚类成功率。

1.5 温和清理与线段拼接（Soft Cleaning & Merging）

(1) 温和清理

使用连通域分析

删除：

面积太小的噪点

横向过宽的块

保留：

合理纵向线条

(2) 拼接断裂线段

对左右两半图像进行形态学闭操作（closing）

补全车道线间的小断裂

结果为：干净、连续、可检测的左右两条车道线。

1.6 骨架化 + DBSCAN 聚类

处理流程：

Canny 边缘检测

闭操作

骨架化（Thinning） → 得到单像素宽度的主干线

DBSCAN 聚类 → 自动区分左右车道线

不需要指定簇数

噪声（label = -1）自动剔除

小簇（<50点）过滤

Silhouette Score 评估聚类质量

优点：

抗噪能力强

能适应车道线断裂、光照变化

自动找到左右两条主干线

1.7 曲线拟合（Curve Fitting）

对每个簇的点集进行二次多项式拟合：

x = a·y² + b·y + c

为什么不用 y = f(x)？

在弯道可能出现“一条 y 对应多个 x”的多值问题

而 y 始终单调递增，更适合建模 BEV 中的纵向车道结构

输出：

左车道拟合曲线

右车道拟合曲线

在 BEV 图像上叠加可视化

1.8 坐标生成、偏向角、中心偏移量（Lane Point Generation）

完成拟合后，需要将连续曲线离散化成像素点：

从 y = ymin → ymax 均匀采样

根据函数计算 x

得到结构化点集（按 y 递增排序）

额外计算：

两条车道线的方向角（朝向角）

左右车道线与中线之间的距离

小车相对中心线的偏移量

所有结果存储到 JSON，用于控制模块。

未来可通过真实赛道标定：

将 pixel → meter 转换为真实距离

2. 结果分析（Results）

基于 111 张模拟器图像的测试结果：

a) 时间性能

平均耗时：0.015 s/帧

标准差：0.00725 s

最快：0.009 s

最慢：0.0397 s

➡ 支持高刷新率实时控制

b) 鲁棒性

识别准确率：93.69%

c) 混淆情况

在仅看到一条车道线时仍可保持区域识别

左右簇未混淆比例：98.19%

聚类数量：

平均：2.1

标准差：0.3

最优：2

最差：3

3. 结论（Conclusion）

整体算法在模拟器测试中表现稳定：

实时性强：支持高频控制循环

准确率高：93.69%

左右车道分离效果良好：98.19% 未混淆

能够处理车道丢失、部分遮挡等情况

适用于真实环境和模拟环境的车道检测任务。

4. 展望（Perspectives）

当前方案的局限：

对光照（阴影、反光）较敏感

参数（阈值、ROI等）依赖人工调整

在复杂环境的泛化能力有限

未来改进方向：

引入深度学习车道检测模型

社区成熟模型丰富

更强的抗光照、抗噪声鲁棒性

泛化能力明显优于传统方法

自适应阈值 / 滤波方法（但计算量较大）

深度模型 + 当前几何管线结合

进一步提高稳定性与实时性能

GitLab使用教程

GitLab使用教程

在将DonkeyCar项目推进到小组进行整合时，要用到学校的GitLab，所以今天就来顺便学习一下Git的基本用法。

基本流程

一、配置SSH 链接GitLab

  1. 首先使用学校邮箱创建个人账户 我的邮箱是lixi.wang@imt-atlantique.net 用户名是l24wang 这步很简单就暂且不提。

  2. 第二步，配置SSH key 这步对于每个账号只需要配置一次： ssh-keygen -t ed25519 -C "你的邮箱"
    ✅ -t ed25519 指定加密算法
    ✅ -C 是备注信息（写邮箱）
    ⛔ 不需要设置密码

  3. 对于Windows：启动 ssh-agent 并载入 key：

    Start-Service ssh-agent
    ssh-add $env:USERPROFILE\.ssh\id_ed25519

  4. 将key添加到GitLab: 复制在终端中生成的key，然后在网站的key选项那里手动添加。

  5. 测试key连接是否成功 ssh -T git@gitlab.imt-atlantique.fr
    成功显示： Welcome to GitLab, @l24wang!

二、克隆 GitLab 仓库到本地

  1. 先 cd 到想要项目到本地的目的文件夹里
  2. 用git clone git@gitlab.imt-atlantique.fr:仓库路径.git
    我这里是 git clone git@gitlab.imt-atlantique.fr:z23mahmo/2025-2026-ia-racing.git
    git@…是使用 SSH 方式下载的意思

三、把自己的文件加入仓库（本地）

复制粘贴你的文件到仓库文件夹里，注意现在你的文件还没有更新到线上仓库和别人共享，只能在本地查看。

四、把自己的文件更新到线上仓库

git add .
git commit -m "Add my code"
git push origin main

DonkeyCar 学习日记

Day 1 - 霍夫变换车道线检测方法

🚗 思路流程整理

  1. 将 RGB 图像转换为 HLS 颜色空间，提取白色可能区域
  2. 阈值过滤后得到黑白二值图
  3. 设计 ROI 区域避免无关干扰
  4. 使用 Canny 边缘检测提取道路边缘
  5. 使用 HoughLinesP 检测直线
  6. 根据斜率判断左右车道线
  7. 根据路况选择拟合方式（直线/曲线）

🔍 Canny 边缘检测

参数说明：

参数 含义
image 输入图像（单通道灰度图）
threshold1 Canny低阈值
threshold2 Canny高阈值
apertureSize Sobel算子大小（3/5/7）
L2gradient True 更精确但更慢

📐 Hough 直线检测

作用：霍夫变换将边缘点转为参数空间，用投票机制拟合出直线。

➗ 区分左右车道线逻辑

通过霍夫拟合出的直线斜率：

  • 斜率 < 0 → 左车道线
  • 斜率 > 0 → 右车道线

🛣️ 判断直道还是弯道

类型 特征
直道 霍夫线段斜率接近
弯道 拟合出的线段呈弧线，斜率变化大

若是弯道，需要使用二次曲线拟合。

❗ 当前遇到的问题

  1. 很难准确区分真实白线与光照高亮区域
  2. 强反光区域容易被当成车道线
  3. 二值图噪声较多，导致霍夫检测不稳定
  4. 由于前一步二值化不够稳定，Canny 检测受噪声干扰较大

✅ 下一步我计划尝试：

  • 改进二值化方法，尝试转换到灰度空间
  • 修改ROI区域的形状使其更贴合实际形状
  • 尝试使用clustering，K聚簇，DBSCAN算法来进行曲线拟合

DonkeyCar 学习日记

Day 2 - 霍夫变换车道线检测方法若干问题的改进

霍夫变换改进

  1. 改变ROI从梯形到五边形
  2. 预处理图像从HLS二值转换变成灰度值转换
  3. 用K-means和DBSCAN算法处理霍夫变换得到的线段集
  4. 增加BEV鸟瞰图变换
    def bev_perspective(img): h, w = img.shape[:2] src_points = np.float32([ [w * 0.05, h * 0.7], [w * 0.95, h * 0.7], [w * 0.70, h * 0.45], [w * 0.30, h * 0.45] ]) dst_points = np.float32([ [w * 0.25, h], [w * 0.75, h], [w * 0.75, 0], [w * 0.25, 0] ]) M = cv2.getPerspectiveTransform(src_points, dst_points) return cv2.warpPerspective(img, M, (w, h))
    本函数把原图中的梯形映射成矩形。

车道线相关Python内容

  1. import pathlib as Path 可以让我方便地自动处理不同操作系统的路径格式（如 Windows 的 \ 和 Linux 的 /），即判断指定路径下的文件是否存在。
  2. Path对象用来创建和保存文件非常方便 在我的程序中使用Path对象存放输入输出图片。
  3. gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)把图片从RGB换成灰度空间的函数。
  4. blur = cv2.GaussianBlur(gray, (GAUSS_KSIZE, GAUSS_KSIZE), 0)对图片进行高斯滤波。
  5. v = np.median(blur)median函数是对矩阵求中位数。
  6. 将形如(x,y)的点集的x 和y提取出来的方法：首先有点集pointspoints = [(100, 720), (110, 680), (120, 640), (130, 600)]
    对points进行处理：

pts = np.array(points) ys = pts[:, 1] xs = pts[:, 0] :代表矩阵里的所有元素 这里是所有行，0 和 1代表矩阵的第0列和第1列。
