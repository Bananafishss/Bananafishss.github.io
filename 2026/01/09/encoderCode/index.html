微处理器 PWM控制机械指针

指针角度和计数器计数之间的转换的两个问题：

  1. 角度超出180
  2. 系统上电时初始角度不一致

指针旋转角度和马达计数器之间的转换

“一个计数对应指针轴多少度”函数

‘’’c
static inline float degPerDriveCountOnShaft() {
return 360.0f / (float)ONE_REV_COUNTS / GEAR_RATIO;
}
‘’’
指针轴每个 count 转的角度 = 360 / 12 / 52.734

wrap180 函数是将角度限制到-180 180

角度 → 计数上面的逆函数

首先我们有 不含校准 offset
‘’’
static inline long shaftDegToDriveCount(float deg) {
return lroundf(deg / degPerDriveCountOnShaft());
}//count = 角度 / (每个 count 对应的角度)
‘’’
然后 改进版 角度 → 目标计数（含校准 offset:
‘’’c
long expectedDegToTargetCount(float expDeg) {
// expDeg == count*degPer + offset => count = (expDeg - offset)/degPer
float pure = wrap180(expDeg - calibOffsetDeg);
return lroundf(pure / degPerDriveCountOnShaft());
}//系统上电时，编码器的 0 点 ≠ 指针表盘的 0 度,所以引入了 calibOffsetDeg（校准角偏置）
‘’’

  1. 先expDeg - calibOffsetDeg：把“表盘角度”换到“编码器坐标系”

  2. 然后用wrap180：避免角度跨 ±180 产生跳变

  3. 最后使用没校准的函数 除以 degPerDriveCountOnShaft() 得到 count（整数）

驱动马达旋转到目标位置 也就是让count和目标计数targetCount之间的差值err为零

  1. 先读编码器的计数得到当前count
  2. 根据当前count 和目标count 之间的误差err 控制电机旋转 直到误差为0。

读取编码器“当前位置”的count

‘’’c
Encoder enc(PIN_ENC_A, PIN_ENC_B);
//这个函数就是当前电机（或驱动轮）的“角度位置”用计数表示。
long readCountDrive() {
long c = enc.read();//读出增量编码器累计计数（count）
if (INVERT_ENCODER) c = -c; //如果转向/计数方向反了，就翻转
return c;
}
‘’’

设置 目标计数targetCount

目标计数targetCount根据温度传感器的当前值得到，付给变量long targetCountDrive 。

误差计算

long err = targetCountDrive - c;
err > 0：说明还没转够，要往正方向转

err < 0：转过头了，要往反方向转

err = 0：刚好到位
最经典的闭环思路：控制误差趋近于 0

电机驱动

‘’’c
void setMotor(bool dirHigh, int pwm) {
if (INVERT_MOTOR_DIR) dirHigh = !dirHigh;
digitalWrite(PIN_DIR, dirHigh ? HIGH : LOW);
analogWrite(PIN_EN, pwm);
}
‘’’
PIN_DIR：方向（HIGH/LOW）

PIN_EN：PWM（0~255，越大越用力）

INVERT_MOTOR_DIR：如果方向反了就翻转 由于无法确定旋转方向 所以在这里设置一个检验语句用来纠错

电机旋转矫正 控制律

  1. 超时保护（防止卡住一直转）
    ‘’’c
    if (mstate != M_IDLE && now - moveStartMs > TIMEOUT_MS) {
    mstate = M_IDLE;
    stopMotor();
    }
    ‘’’
    如果 7 秒还没到位，强制停机
  2. “稳定到位”判定
    误差为 0 稳定 500ms 才算到位：
    ‘’’c
    if (err == 0) {
    if (stableZeroT0 == 0) stableZeroT0 = now;
    } else stableZeroT0 = 0;

if (stableZeroT0 != 0 && now - stableZeroT0 >= STABLE_MS_ZERO) {
mstate = M_IDLE; stopMotor();
}

‘’’
或者允许 ±1 误差稳定 800ms 也算到位（fallback）：
‘’’c
if (labs(err) == 1) {
if (stableOneT0 == 0) stableOneT0 = now;
} else stableOneT0 = 0;

else if (ALLOW_FALLBACK_PLUSMINUS1 && stableOneT0 != 0 && now - stableOneT0 >= STABLE_MS_ONE) {
mstate = M_IDLE; stopMotor();
}
‘’’
3. 状态机：远的时候快跑，近的时候慢跑，最后轻推
定义了状态: enum MotionState { M_IDLE, M_COARSE_FAST, M_COARSE_SLOW, M_NUDGE_OFF, M_NUDGE_ON };
距离判断（看误差绝对值 aerr）

aerr >= FAR_ZONE(8)：离得远 → M_COARSE_FAST（大力）

3~7：中等距离 → M_COARSE_SLOW（小力）

<= FINE_ZONE(2)：很近 → 切到 NUDGE（一下一下点动）
‘’’c
if (aerr <= FINE_ZONE) { … NUDGE … }
else if (aerr < FAR_ZONE) { mstate = M_COARSE_SLOW; }
else { mstate = M_COARSE_FAST; }
‘’’
4. 真正的控制律：P 控制 + PWM 限幅
1）粗调（coarse）：比例控制 u = Kp * err
‘’’c
float u = KP_COARSE * (float)err;
bool dirHigh = (u >= 0);
int pwm = (int)fabs(u);
‘’’
KP_COARSE=7.0

pwm 跟误差成正比：误差大 → PWM 大
2）PWM 限制（防止太大/太小）

FAST 模式：
if (pwm > PWM_COARSE_MAX_FAST) pwm = PWM_COARSE_MAX_FAST; // 130
if (pwm > 0 && pwm < PWM_COARSE_MIN_FAST) pwm = PWM_COARSE_MIN_FAST; // 60

SLOW 模式：
if (pwm > PWM_COARSE_MAX_SLOW) pwm = PWM_COARSE_MAX_SLOW; // 100
if (pwm > 0 && pwm < PWM_COARSE_MIN_SLOW) pwm = PWM_COARSE_MIN_SLOW; // 55

最小 PWM 是为了克服静摩擦（不然给太小动不了）。
最大 PWM 是为了别冲太猛、别过冲。
5）最后的精调：NUDGE 点动（避免来回震）

当误差很小（<=2 counts），它不用 P 控制了，而是：

先停一会儿 NUDGE_OFF_MS=160ms

再用固定 PWM=55 推一下

推的时间很短：

误差 <=1：推 15ms

否则推 35ms

unsigned long onMs = (aerr <= 1) ? NUDGE_ON_MS_FINE : NUDGE_ON_MS_COARSE;
bool dirHigh = (err > 0);
setMotor(dirHigh, PWM_NUDGE);
…
}
‘’’
